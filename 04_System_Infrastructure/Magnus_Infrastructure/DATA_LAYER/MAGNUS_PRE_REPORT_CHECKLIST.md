# MAGNUS PRE-REPORT CHECKLIST

> No dossier ships without completing this checklist.
> This checklist supplements the Playbook's Non-Negotiables (Section 6) with data integrity and anti-backlash gates.

**Status:** MANDATORY — All Engagements
**Cross-Reference:** MAGNUS_MASTER_PLAYBOOK.md → Section 6 (Non-Negotiables)

---

## Before Sending Any Dossier

### Data Integrity (New — Data Layer Gates)
- [ ] Every financial claim has a corresponding entry in the **Data Register**
- [ ] Every major number has confidence tag assigned (A/B/C/U → VERIFIED/ESTIMATED/PROXY/HYPOTHESIS)
- [ ] No HYPOTHESIS-level numbers presented as facts in the report
- [ ] Claims using PIFR are explicitly labeled "Peer-Implied" with methodology shown
- [ ] Every competitor claim verified with independent search (Lesson L8)
- [ ] Calculation logic documented for any estimated financial figure

### Playbook Non-Negotiables (Existing — From Section 6)
- [ ] At least 7 distinct signals with confidence grades
- [ ] At least 3 contradictions identified and graded
- [ ] 10 interrogation questions with "red flag vs good answer"
- [ ] "What Would Change My Mind" for every A/B finding
- [ ] Explicit blind spots section
- [ ] Source citations for every claim
- [ ] NDI Claim Ledger with all company claims tracked
- [ ] MSRI Composition Appendix with full formula
- [ ] Prediction Registry with 5+ falsifiable predictions
- [ ] Catalyst Monitor with targets and failure signals
- [ ] Market Snapshot with sourced sector data
- [ ] Methodology section explaining the process

### Anti-Backlash Checks (Learned from GFC/PASHA)
- [ ] Executive Summary leads with strengths, then risks (Lesson L12)
- [ ] No absolute claims ("sole producer", "only company") without multi-source verification
- [ ] Competitor comparisons use neutral, benchmark-style language
- [ ] Every major finding includes a verification path (what internal data would confirm)
- [ ] No specific PKR amount without documented calculation trail
- [ ] Self-corrections from previous versions are visible (Lesson L15)

### Tone & Language
- [ ] Diagnostic tone, not accusatory
- [ ] Bold claims softened with "based on available public signals"
- [ ] Governance/ownership language is factual, not judgmental
- [ ] Every criticism paired with "What Would Change My Mind"

### Documentation
- [ ] Client Data Log updated with all sources used
- [ ] Data Register updated with any new data points introduced
- [ ] Lessons Learned (Playbook Section 7) reviewed for applicability
- [ ] Prediction Registry updated with any resolved predictions

## If a Claim Gets Challenged

1. Open the **Disputed Claims Log** for the client
2. Record: Claim, Source Used, Confidence Grade, Client Objection
3. Check the **Data Register** for the validation trail
4. Check **NDI Claim Ledger** for the original classification
5. Respond with evidence or acknowledge the limitation
6. Update the Data Register and Playbook Lessons if needed

**The system learns from every dispute. Never argue — show the trail.**
