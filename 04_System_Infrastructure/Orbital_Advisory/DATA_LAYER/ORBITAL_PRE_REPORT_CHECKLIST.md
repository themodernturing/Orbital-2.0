# ORBITAL PRE-REPORT CHECKLIST

> No report ships without completing this checklist.
> This is the final gate before delivery.

**Status:** MANDATORY — All Engagements

---

## Before Sending Any Report

### Data Integrity
- [ ] Every number over $100K (or equivalent) has a **Calculation Worksheet**
- [ ] Every major claim has **2+ validation sources** (see Validation Rules)
- [ ] All data points are recorded in the **Master Data Register**
- [ ] Confidence tags are assigned to all key findings (VERIFIED / ESTIMATED / PROXY-BASED / HYPOTHESIS)
- [ ] No HYPOTHESIS-level numbers are presented as facts

### Anti-Backfire Checks
- [ ] No use of the word "Guess" anywhere in the report
- [ ] "Internal Operating Assumption" framing used (not "What you believe")
- [ ] Competitor comparisons use neutral language (benchmarks, not direct attacks)
- [ ] Ownership/governance language is neutral and factual
- [ ] Every major finding includes a **Verification Path** (what internal data would confirm/update)
- [ ] Strategic Response Options are provided for key risks

### Report Structure
- [ ] Protective disclaimer present at the start
- [ ] Evidence Chains included for major findings (Public Signal → Why It Matters → Verification Path)
- [ ] Predictions use proper confidence labels (not "Guess")
- [ ] Report ends with clear next steps / Phase 1 pitch

### Tone & Language
- [ ] Written in simple English — no unnecessary jargon
- [ ] Diagnostic tone (not accusatory)
- [ ] No statements that could be read as personal attacks on leadership
- [ ] Every criticism is paired with a constructive recommendation
- [ ] Bold claims are softened with "based on public signals" or similar qualifiers

### Documentation
- [ ] Client Data Log updated with all sources used
- [ ] Calculation Worksheets saved in client folder
- [ ] Data Register updated with any new data points introduced

## If a Claim Gets Challenged After Delivery

1. Open the **Disputed Claims Log** for the client
2. Record: Claim, Source Used, Client Objection
3. Check the **Data Register** for the validation trail
4. Respond with evidence or acknowledge the limitation
5. Update the Data Register confidence level if needed

**Never argue. Always show the trail.**
